# Security Policy | سیاست امنیتی

## 🇮🇷 فارسی

### گزارش مسائل امنیتی

اگر یک مسئله امنیتی در Kaia Gallery پیدا کردید:

1. **نفرستید** از طریق public issues یا discussions
2. **ایمیل** به security@kaiagoldgallery.ir با جزئیات
3. **موضوع**: "[SECURITY] نام مسئله"
4. **شامل**:
   - توصیف مسئله
   - نسخه تحت تاثیر
   - روش بازتولید
   - پیشنهاد راه‌حل (اختیاری)

ما در بررسی و پاسخ به گزارش‌های امنیتی در عرض 48 ساعت اقدام می‌کنیم.

### حمایت شده‌ی نسخه‌ها

| نسخه | وضعیت | تاریخ پایان |
|------|-------|-----------|
| 1.x | ✅ Supported | 2026-12-31 |
| 2.x | ✅ Supported | 2027-12-31 |

### بهترین‌‌های امنیتی

- استفاده از HTTPS برای تمام ارتباطات
- به‌روزرسانی منظم وابستگی‌ها
- استفاده از JWT برای احراز هویت
- رمزنگاری پسورد‌ها

## 🇬🇧 English

### Reporting Security Issues

If you discover a security vulnerability in Kaia Gallery:

1. **Do NOT** post via public issues or discussions
2. **Email** security@kaiagoldgallery.ir with details
3. **Subject**: "[SECURITY] Issue Name"
4. **Include**:
   - Description of the vulnerability
   - Affected version(s)
   - Steps to reproduce
   - Suggested fix (optional)

We will review and respond to security reports within 48 hours.

### Supported Versions

| Version | Status | End of Life |
|---------|--------|-----------|
| 1.x | ✅ Supported | 2026-12-31 |
| 2.x | ✅ Supported | 2027-12-31 |

### Security Best Practices

- Always use HTTPS for communications
- Keep dependencies updated regularly
- Use JWT for authentication
- Hash passwords securely
- Follow OWASP guidelines
